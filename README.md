# portal-baifucai-resource

## 项目简介

`portal-baifucai-resource` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，仅作为静态页面资源的集合与版本管理用途。

所有页面均为独立文件，彼此不依赖，可单独部署或引用。仓库的主要目的是提供一个整洁、可追溯的页面资源存储空间，便于后续维护与分发。

## 目录结构

```
portal-baifucai-resource/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源（如图片、样式、脚本等）
└── README.md       # 本文件
```

- **pages/**：每个 HTML 页面为一个独立文件，命名清晰，不包含外部依赖。
- **assets/**：若页面间存在共用资源，统一放置于此目录，避免重复。

## 页面归档说明

本仓库中的页面均以“一次归档”的形式保存，即每个页面在发布后保持原样，不进行动态更新。若需更新内容，请创建新版本归档，并更新文件名或版本号以示区分。

页面文件命名建议采用以下格式：

```
<页面标识>-<版本号>.html
```

例如：`dashboard-v1.0.html`、`help-center-v2.1.html`

## 维护说明

- **新增页面**：将 HTML 文件放入 `pages/` 目录，并在本 README 中简要记录新增内容与版本。
- **更新页面**：不建议直接覆盖已有文件，应归档为新版本，保留历史文件。
- **删除页面**：若页面不再需要，请从仓库中移除，并在 README 中注明。
- **分支管理**：默认使用 `main` 分支作为稳定发布分支，开发或测试版本可暂存于其他分支。
